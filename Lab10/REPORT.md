# Звіт до лабораторної роботи №10

## Тема

Узагальнені контейнери та ітератори в Java.

## Варіант

Варіант №3 — система бронювання.

## Короткий опис предметної області

У роботі розглядається система бронювання номерів готелю. У моделі є клієнти, VIP-клієнти, номери різних типів, бронювання, записи про бронювання, розклад бронювань і допоміжні generic-компоненти.

Основні класи предметної області:

- `Customer` — звичайний клієнт;
- `VipCustomer` — VIP-клієнт;
- `Room` — абстрактний номер;
- `StandardRoom`, `FamilyRoom`, `LuxuryRoom` — різні типи номерів;
- `Reservation` — бронювання номера;
- `ReservationRecord<TReservation, TMeta>` — запис про бронювання з метаданими;
- `BookingLink<TCustomer, TResource>` — зв'язок клієнта та номера;
- `BookingSchedule<TReservation>` — розклад бронювань.

## Реалізований контейнер

У роботі реалізовано власний generic-контейнер `ReservationContainer<T>`. Він призначений для зберігання об'єктів системи бронювання, але не прив'язаний до конкретного класу. Завдяки параметру типу `T` контейнер можна використовувати для бронювань, записів бронювання, номерів або інших об'єктів.

## Внутрішнє подання контейнера

Контейнер побудовано на основі внутрішнього масиву `T[] items`. Для збереження кількості фактичних елементів використовується поле `size`, а для контролю місткості — поле `capacity`.

Якщо масив заповнений, викликається метод `grow()`, який створює новий масив більшої місткості та копіює туди наявні елементи.

Стандартні колекції Java не використовуються як основне внутрішнє сховище контейнера.

## Основні методи контейнера

- `add(T item)` — додає елемент у кінець контейнера;
- `get(int index)` — повертає елемент за індексом;
- `set(int index, T item)` — замінює елемент за індексом;
- `remove(int index)` — видаляє елемент і зсуває наступні елементи вліво;
- `contains(T item)` — перевіряє наявність елемента;
- `clear()` — очищує контейнер;
- `size()` — повертає кількість елементів;
- `capacity()` — повертає поточну місткість;
- `isEmpty()` — перевіряє, чи контейнер порожній.

## Реалізація ітератора

У класі `ReservationContainer<T>` реалізовано внутрішній клас `ReservationContainerIterator`, який реалізує інтерфейс `Iterator<T>`.

Ітератор має поле `currentIndex`, метод `hasNext()` для перевірки наявності наступного елемента і метод `next()` для повернення поточного елемента та переходу до наступного.

## Підтримка for-each

Клас `ReservationContainer<T>` реалізує інтерфейс `Iterable<T>` і перевизначає метод `iterator()`. Завдяки цьому контейнер можна обходити у циклі `for-each`.

Приклад:

```java
for (ReservationRecord<? extends Reservation, ?> record : records) {
    System.out.println(record);
}
```

## Результати тестування

Для перевірки роботи контейнера створено тестовий клас `ReservationContainerTest`. У тестах перевіряється:

- додавання елементів;
- отримання елементів за індексом;
- видалення зі зсувом;
- автоматичне розширення масиву;
- робота циклу `for-each`;
- робота з об'єктами `ReservationRecord`;
- обробка неправильних індексів;
- поведінка ітератора після завершення обходу.

Також збережено тести для класів системи бронювання.

## Висновок

У ході роботи було реалізовано власний узагальнений контейнер на основі масиву та власний ітератор для його обходу. Було закріплено практичне використання generics, інтерфейсів `Iterator<T>` і `Iterable<T>`, а також принцип відокремлення логіки зберігання даних від логіки обходу. Реалізація контейнера показала, як на базовому рівні працюють колекції Java до використання готового Collections Framework.
