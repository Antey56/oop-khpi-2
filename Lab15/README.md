# Лабораторна робота №15

## Тема
Синхронізація потоків у Java.

## Предметна область
Система бронювання номерів готелю.

## Варіант
Варіант 3: **Бронювання номерів**. Використані класи `HotelRoom`, `BookingRequest`, `BookingTask`. У проєкті збережена попередня готельна модель: `Room`, `StandardRoom`, `FamilyRoom`, `LuxuryRoom`, `Customer`, `Reservation`.

## Суть роботи
У лабораторній роботі показано проблему спільного змінюваного стану. Кілька потоків одночасно намагаються забронювати номери зі спільного пулу.

Реалізовано два варіанти:

1. `UnsafeHotelRoomPool` — небезпечний варіант без синхронізації.
2. `SynchronizedHotelRoomPool` — коректний варіант із `synchronized`.

## Спільний ресурс
Спільним ресурсом є список доступних номерів і список створених бронювань.

Критична секція: пошук доступного номера, вилучення його зі списку, зміна стану номера і створення бронювання.

## Основні класи
- `BookingRequest` — заявка на бронювання.
- `UnsafeHotelRoomPool` — несинхронізований пул номерів.
- `SynchronizedHotelRoomPool` — синхронізований пул номерів.
- `BookingTask` — задача, яка виконується в окремому потоці.
- `Main` — демонстрація unsafe/safe/sequential сценаріїв.

## Запуск
```bash
mvn test
mvn exec:java -Dexec.mainClass="ua.khpi.oop.lab15.Main"
```

Або через IDE: запустити клас `ua.khpi.oop.lab15.Main`.
