# Звіт до лабораторної роботи №12

## Тема
Stream API в Java.

## Варіант
Варіант 6 — система бронювання. Основні класи предметної області: `Reservation`, `Customer`, `Room`.

## Мета роботи
Метою роботи було набуття практичних навичок використання Stream API для декларативної обробки колекцій об'єктів: фільтрації, перетворення, сортування, групування, підрахунку та формування зведень.

## Опис предметної області
Предметна область — готельна система бронювання. У системі є клієнти, номери різних типів та бронювання. Кожне бронювання має ідентифікатор, клієнта, номер, дату заїзду, дату виїзду, кількість ночей, активний статус і підсумкову вартість.

## Зв'язок із лабораторною роботою №11
У лабораторній роботі №11 використовувався `HotelBookingRegistry`, який зберігав бронювання у стандартних колекціях Java. У лабораторній роботі №12 ця модель не замінюється, а розширюється сервісом `BookingAnalytics`, який обробляє підготовлену колекцію бронювань через Stream API.

## Реалізовані stream-операції

### 1. Фільтрація
Метод `findActiveReservations()` використовує `filter()` для відбору лише активних бронювань.

Метод `findReservationsInDateRange()` відбирає бронювання, які потрапляють у заданий діапазон дат, а потім сортує їх за датою заїзду.

### 2. Перетворення
Метод `getCustomerNamesSorted()` перетворює бронювання на імена клієнтів через `map()`, усуває дублікати через `distinct()` і сортує результат.

Метод `buildShortReservationLines()` перетворює об'єкти бронювання на короткі текстові рядки для звіту.

### 3. Сортування
Метод `getReservationsSortedByTotalCostDescending()` використовує `Comparator.comparingDouble()` для сортування бронювань за вартістю у спадному порядку.

### 4. Агрегування
Метод `countActiveReservations()` рахує кількість активних бронювань.

Метод `calculateTotalRevenue()` використовує `mapToDouble()` і `sum()` для підрахунку загальної виручки лише за активними бронюваннями.

Метод `findMostExpensiveReservation()` використовує `max()` і повертає `Optional<Reservation>`, тому що список бронювань може бути порожнім.

### 5. Групування та статистика
Метод `groupReservationsByRoomType()` групує бронювання за типом кімнати через `Collectors.groupingBy()`.

Метод `countReservationsByRoomType()` поєднує `groupingBy()` та `Collectors.counting()`.

Метод `getTotalCostStatistics()` використовує `Collectors.summarizingDouble()` для отримання кількості, суми, мінімуму, максимуму та середнього значення.

### 6. Побудова текстового звіту
Метод `buildRoomTypeSummaryReport()` формує текстове зведення за типами кімнат. Для цього використано обхід `entrySet().stream()` і `Collectors.joining()`.

## Порівняння імперативного та декларативного підходів
Для демонстрації додаткового завдання реалізовано два методи:
- `findByRoomTypeImperative()` — традиційний цикл `for`;
- `findByRoomTypeStream()` — декларативний варіант через `stream()`, `filter()` і `collect()`.

Обидва методи дають однаковий результат. Stream-варіант коротший і краще передає намір: потрібно не вручну керувати циклом, а відібрати бронювання за типом номера.

## Тестування
У класі `BookingAnalyticsTest` перевірено:
- відбір активних бронювань;
- пошук за діапазоном дат;
- сортування імен клієнтів;
- групування та підрахунок за типом кімнати;
- підрахунок активної виручки;
- пошук найдорожчого бронювання;
- статистичне зведення;
- однаковість результатів імперативного й stream-підходу;
- роботу з порожньою колекцією.

## Висновок
У ході роботи було реалізовано обробку колекцій готельних бронювань за допомогою Stream API. Було показано використання базових і складніших stream-операцій: `filter()`, `map()`, `sorted()`, `distinct()`, `collect()`, `groupingBy()`, `counting()`, `sum()`, `max()` та `summarizingDouble()`. Stream API дозволив описати обробку даних більш декларативно та читабельно, особливо для фільтрації, групування і побудови звітів.
