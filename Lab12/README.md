# Лабораторна робота №12

## Тема
Stream API в Java.

## Варіант
Варіант 6 — система бронювання: `Reservation`, `Customer`, `Room`.

## Опис
Проєкт продовжує модель готельного бронювання з лабораторної роботи №11. У ЛР11 основна увага була на стандартних колекціях Java, а в ЛР12 ці колекції обробляються через Stream API.

## Основні класи
- `Customer`, `VipCustomer` — клієнти готелю.
- `Room`, `StandardRoom`, `FamilyRoom`, `LuxuryRoom` — номери різних типів.
- `Reservation` — бронювання з датами, статусом і вартістю.
- `HotelBookingRegistry` — реєстр бронювань на базі стандартних колекцій.
- `BookingAnalytics` — сервіс аналітики зі Stream API.
- `Main` — демонстраційна програма.

## Stream-операції
У класі `BookingAnalytics` реалізовано:
- `filter()` — активні бронювання, бронювання за діапазоном дат;
- `map()` — отримання імен клієнтів і коротких рядків звіту;
- `sorted()` / `Comparator` — сортування за датою, ціною, іменем;
- `distinct()` — унікальні імена клієнтів;
- `collect()` — збирання результатів у `List`, `Set`, `Map`;
- `groupingBy()` і `counting()` — групування та підрахунок за типом кімнати;
- `sum()` — підрахунок загальної активної виручки;
- `max()` і `Optional` — пошук найдорожчого бронювання;
- `summarizingDouble()` — статистика вартості.
