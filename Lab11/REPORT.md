Що було змінено порівняно з лабораторною роботою №9

У лабораторній роботі №9 основна увага приділялася використанню узагальнень (Generics), створенню узагальнених класів та методів для системи бронювання номерів у готелі.

У лабораторній роботі №10 функціональність системи була змінена та розширена відповідно до теми роботи «Узагальнені контейнери та ітератори».

Було виконано такі зміни:

На основі предметної області попередньої лабораторної роботи залишено основні сутності:
Room;
Customer;
Reservation.
Додано власний узагальнений контейнер:
ReservationContainer<T>

який призначений для зберігання об'єктів бронювання та інших типів даних.

Для зберігання елементів у контейнері використано власний динамічний масив:
private T[] items;

з автоматичним збільшенням розміру при переповненні.

Реалізовано базові операції контейнера:
додавання елементів (add);
отримання елементів (get);
видалення елементів (remove);
визначення кількості елементів (size);
перевірка на порожність (isEmpty).
Реалізовано власний ітератор:
ReservationIterator

що забезпечує послідовний обхід елементів контейнера.

Контейнер реалізує інтерфейс:
Iterable<T>

що дозволяє використовувати цикл:

for (Reservation reservation : reservations)

без необхідності ручного керування індексами.

На відміну від лабораторної роботи №9, де використовувалися generic-класи предметної області (BookingLink, BookingSchedule, ReservationRecord), у лабораторній роботі №10 основний акцент зроблено на створенні власної контейнерної структури даних.
Додано автоматичне збільшення місткості контейнера при заповненні внутрішнього масиву.
Реалізовано модульні тести для перевірки:
роботи контейнера;
додавання елементів;
видалення елементів;
роботи ітератора;
роботи циклу for-each;
автоматичного розширення контейнера.
Структуру проєкту було спрощено. Частина допоміжних generic-класів із лабораторної роботи №9 не використовувалась, оскільки вони не були необхідні для реалізації контейнера та ітератора.

Таким чином, якщо в лабораторній роботі №9 головною метою було вивчення механізму Generics, то в лабораторній роботі №10 основна увага приділялася створенню власного узагальненого контейнера даних, реалізації ітератора та організації обходу колекції через механізм Iterable<T>. Це дозволило поглибити розуміння принципів роботи контейнерів та внутрішньої організації структур даних у Java.
