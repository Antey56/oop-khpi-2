
# Звіт до лабораторної роботи №11

## Тема

Collections Framework в Java.

## Мета роботи

Метою роботи є формування практичних навичок використання стандартних колекцій Java, вибору відповідного типу колекції для конкретного завдання, організації зберігання наборів об'єктів та виконання базових операцій над ними.

## Предметна область

У роботі використано предметну область системи бронювання готельних номерів, яка була сформована в попередніх лабораторних роботах. Основними сутностями є `Customer`, `VipCustomer`, `Room`, `StandardRoom`, `FamilyRoom`, `LuxuryRoom`, `Reservation` та допоміжні класи бронювання.

## Архітектурне рішення

Для лабораторної роботи №11 було додано сервісний клас `HotelBookingRegistry`. Він інкапсулює стандартні колекції Java та надає прикладні методи для роботи з бронюваннями. Логіка предметної області відокремлена від логіки зберігання наборів даних.

## Використані колекції

1. `ArrayList<Reservation>` використано для зберігання бронювань у порядку додавання. Такий вибір доречний, бо для демонстрації потрібно зберігати послідовність і виконувати послідовний перегляд.
2. `HashMap<String, Reservation>` використано для швидкого пошуку бронювання за ідентифікатором. Ключем є `reservationId`.
3. `HashSet<Customer>` використано для зберігання унікальних клієнтів. Коректність роботи множини забезпечується реалізацією `equals()` та `hashCode()` у класі `Customer`.
4. `ArrayDeque<Reservation>` використано як чергу обробки бронювань. Вона дозволяє додавати елементи через `offer()` та отримувати наступне бронювання через `poll()`.

## Основні методи сервісу

- `addReservation()` додає бронювання одразу до списку, мапи, множини клієнтів і черги обробки.
- `findById()` виконує швидкий пошук через `HashMap`.
- `removeReservation()` видаляє бронювання і підтримує узгодженість усіх внутрішніх колекцій.
- `cancelReservation()` змінює стан бронювання.
- `findByCustomer()` і `findByRoomType()` виконують пошук за ознакою.
- `getReservationsSortedByTotalCostDescending()` демонструє сортування через `Comparator`.
- `buildReservationReport()` демонструє обхід `Map` через `entrySet()`.

## Обхід колекцій

У демонстраційній програмі показано обхід списку через `for-each`, обхід списку через `Iterator`, обхід мапи через `entrySet()` та обробку черги через `poll()`.

## Тестування

Модульні тести перевіряють початковий стан реєстру, додавання, пошук за ключем, заборону дублювання `reservationId`, роботу множини унікальних клієнтів, видалення, чергу обробки, пошук за клієнтом і типом кімнати, скасування бронювання та сортування.

## Висновок

У ході роботи було реалізовано сервіс для керування бронюваннями на основі стандартних колекцій Java. Було показано, що різні колекції виконують різні ролі: `List` зберігає порядок, `Map` забезпечує швидкий доступ за ключем, `Set` прибирає дублікати, а `Queue` моделює чергу обробки. Робота продовжує попередню лабораторну, але замість самописного контейнера використовує інструменти Collections Framework.
