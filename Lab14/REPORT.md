
# Звіт до лабораторної роботи №14

## Тема

Базова багатопотоковість у Java.

## Предметна область

У роботі використано систему бронювання готельних номерів. Предметна модель продовжує попередні лабораторні роботи та містить клієнтів, номери різних типів і бронювання.

## Реалізовані потоки

У програмі реалізовано кілька незалежних задач:

1. `ReservationCheckThread` — клас-нащадок `Thread`, який перевіряє заявки бронювання.
2. `ConfirmationTask` — клас, який реалізує `Runnable` і формує підтвердження бронювань.
3. `OccupancyReportTask` — клас, який реалізує `Runnable` і будує звіт про завантаження номерів.

Потоки запускаються методом `start()`. Головний потік очікує завершення всіх робочих потоків через `join()`. Для імітації поетапного виконання використано `Thread.sleep()`.

## Особливості виконання

Порядок проміжних повідомлень у консолі може змінюватися між запусками, оскільки планувальник потоків не гарантує однаковий порядок виконання. Правильність програми визначається не порядком повідомлень, а тим, що фінальні результати читаються тільки після `join()`.

## Тестування

Модульні тести перевіряють детерміновану логіку:

- перевірку активності бронювання;
- формування підтвердження;
- побудову звіту про завантаження;
- роботу `Runnable`-задачі через прямий виклик `run()` без залежності від планувальника потоків.

## Висновок

Було реалізовано базову багатопотокову програму для системи бронювання. Було показано два способи створення потоків: через наслідування від `Thread` і через реалізацію `Runnable`. Було використано `start()`, `sleep()`, іменування потоків та `join()` для контрольованого завершення програми.
